# ねこちゃんアプリ (pet-app)

> 7歳の女の子のための、ペットでウェイトを上げる仮想ペットアプリ。
> このファイルは企画メモ兼・開発仕様です。Claude Code にそのまま渡して使います。

---

## このアプリは何か / なぜ作るのか

ペットを飼いたいけど、アパートでは飼えないという、7歳の女の子の願いから生まれたアプリ。

本人もすでに競合を整理していて、立ち位置がはっきりしている:

- リアルなペット → アパートでは飼えない
- ロボットペット → 高い・見た目が固定で選べない
- たまごっち → 古い・ちょっと違う

→ **このアプリならではの価値**:
**アパートでも飼えて、見た目を自由に着せ替えられて、ちゃんとなついてくれるペット**
この一文がこのプロジェクトの軸。迷ったらここに立ち返る。

## だれが使う

- 7歳の女の子（ひとり）
- まずは本人が遊ぶための個人アプリ。将来みんなに開くかは後で考える。
- UIは7歳が直感で触れることを最優先。文字は少なく、大きく、タップしやすく。

## プラットフォーム

**まずはWebアプリ（PWA）として作る**

- ブラウザで開発 → iPhoneのSafariでホーム画面に追加すると、アイコンが並んで全画面起動。ほぼネイティブアプリっぽく見える。
- Mac / Xcode / Apple Developer年会費 / 審査 すべて不要。
- データは端末側に保存（後述のlocalStorage）。サーバー不要。
- App Store配信が必要になったら、その時点でネイティブ化を検討する（設計の考え方は同じ）。

---

## いちばん大事なこと（魔法の瞬間）

このアプリの心臓は、着せ替えでもクイズでもなく、こう:

> **なでる → ハートが出る → なついてくる**

これを「可愛い・気持ちいい」と思ってもらえれば成功の合図。
**まずフェーズ1でこの体験だけを完成させる。** 着せ替えもクイズもあとからで十分。

---

## 全体ロードマップ（フェーズ分け）

いきなり全部作らない。1フェーズずつ完成させて、その都度iPhone実機で確認してから次へ進む。

| フェーズ | 内容 | 状態 |
|---|---|---|
| 0 | 紙とペンで決める（ペットの見た目・名前・画面）。娘から発注書を親が制作。 | ◯ |
| **1（MVP）** | **なでる→ハート→なつき度メーターだけを完成させる。** | **← 今ここを作る** |
| 2 | 着せ替え（4〜6個、最初は全部無料で着せられる） | あとで |
| 3 | クイズ（国語・算数）→ポイント→ショップで服を解放 | あとで |
| 4 | 効果音・アニメ強化・ペット・服追加などの必要ならネイティブ化 | あとで |

## フェーズ1 詳細仕様（いちばん最初の制作ターゲット）

### 画面

- 画面の真ん中にペットを1匹。
- 画面上部に「なつき度」メーター（ハートが少しずつ満ちていく等）。
- それ以外は極力ない。シンプルに。

### 動き

1. **アイドル状態**: 何もしなくてもペットがちょこちょこ（呼吸するように小さく動いてたり、生きてる感じ）
2. **なでる**: ペットをタップ、またはなでる（ドラッグ）と…
   - ハートがふわっと数個ポンと飛び出す（ハプティック）。
   - ペットがちょっとだけプルプルと揺れる／表情が笑顔になる。
   - なつき度が少し上がる。
3. **なつき度が上がると**: ペットの反応が少し豊かになる（任意。最初はメーターが満ちていくだけでも可）。

### 保存（localStorage）

- なつき度の値を端末に保存する。
- 次にアプリを開いたとき、前回のなつき度を覚えている。
- 保存キー例: `petApp.affection`

### この時点で入れないもの

- 着せ替え・クイズ・ポイント・ショップ・複数ペット → すべてフェーズ2以降。
- フェーズ1は「なでて、なついていく」を伝えるだけで→完成とする。

## 技術メモ（Claude Code向けの目安）

- 単一ページのWebアプリ。HTML / CSS / JavaScript（またはReact）で軽く作る。フレームワークは凝らなくていい。
- ペットの絵は、まずは絵文字や簡単な図形（SVGで仮置き）でOK。後で娘の描いた絵に差し替える。
- ハートのハプティックは、CSSアニメーション or canvasを軽く触ってみる。
- 保存はブラウザの `localStorage` のみ。サーバー・アカウント不要。
- 「ホーム画面に追加」で全画面になるよう、PWAの最低限（manifest, アイコン）を入れておくと本物っぽくなる。

## まだ決めていないこと（未定のままでOK）

- ペットの種類（犬・猫・架空の生き物など）
- ペットの名前（例:ポメなど、娘から決める）
- 色や世界観のトーン

→ 未定でも開発は始められる。仮の見た目で作って、後から差し替える。
